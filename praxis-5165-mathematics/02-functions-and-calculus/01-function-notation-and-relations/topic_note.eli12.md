Think of a vending machine. You press a button, and a snack drops out. In math, a relation is a set of ordered pairs mapping an input value to an output value. It is just a list showing which buttons connect to which snacks. 

But what if you press button A1 and get potato chips today, but tomorrow you press A1 and get a pair of dirty socks? That is a broken machine! To fix this, we use functions. A function is a specific type of relation where each input value maps to exactly one output value. This means our math is predictable. If we put in an input of 5, we need to know exactly what output we will get every single time. 

Learning how these math machines work—what goes into them, what comes out of them, and the rules they follow—is like learning how to program your own video game!

## Anatomical Components: Domain and Range

To understand a function, we have to look at what it allows us to do. 
*   Think of the coins a vending machine accepts. In math, the set of all valid input values for a relation is called the domain. 
*   Think of the total menu of snacks the machine can give you. The set of all possible output values for a relation is called the range.

If you are looking at a picture of a graph, you can find these sets by looking at the directions the lines stretch. The domain of a function can be found by identifying the horizontal spread of the function's graph on the Cartesian coordinate system (how far it goes from left to right). The range of a function can be found by identifying the vertical spread of the function's graph on the Cartesian coordinate system (how far it goes up and down).

There is also a really cool trick if we play the machine backward! If we swap the inputs and outputs, we create an "inverse" relation. The neat part is that the domain of a relation is exactly equal to the range of the inverse of that specific relation. At the same time, the range of a relation is exactly equal to the domain of the inverse of that specific relation! 

## Identifying Functions Across Representations

You will see relations written in lots of different ways. Your job is to test them to see if they are perfectly predictable—meaning they earn the special title of "function."

### Tables and Mapping Diagrams
Sometimes, you will just look at lists of numbers. A table of values represents a function if no single input value is paired with multiple different output values. If you see the number 3 in the "input" column giving you a 4, and later the same number 3 gives you an 8, the rule is broken!

We can also draw this with arrows. In a mapping diagram, a relation is a function if exactly one arrow originates from each element in the domain. It is perfectly fine if two different buttons give you the same snack (like two arrows pointing to "potato chips"). But a single input button cannot shoot out arrows to two totally different snacks!

### Graphical Analysis
When you are looking at a drawing of a math line, there is an easy visual trick. The Vertical Line Test states that a graph represents a function if no vertical line intersects the graph more than once. Imagine sliding a straight, straight-up-and-down pencil across your paper from left to right. If your pencil ever touches the drawing in two places at the exact same time, that means one input just gave you two outputs!

There is one line that fails this test completely. The vertical line equation x = c, where c is a constant, does not represent a function of x. Because it is perfectly straight up-and-down, your pencil would touch it everywhere all at once!

### Algebraic Equations
Sometimes, you have to test a written math equation to see if it acts like a function. An equation represents y as a function of x if the equation can be solved to yield a unique y-value for every valid x-value. 

Watch out for equations that have squared letters! Equations containing an even power of y generally do not represent y as a function of x. Let's prove this by looking at a circle equation: x^2 + y^2 = 25.

Step 1: Write down the equation.
x^2 + y^2 = 25

Step 2: Try to get y^2 all by itself. Subtract x^2 from both sides of the equals sign.
y^2 = 25 - x^2

Step 3: Solve for y by taking the square root of both sides.
y = square root of (25 - x^2) AND y = negative square root of (25 - x^2)

Because taking a square root gives both a positive and negative answer, plugging in one input for x (like the number 3) gives you two totally different outputs for y (a 4 and a -4). It is not a function!

## The Grammar of Mathematics: Function Notation

To write these rules quickly, mathematicians use a special tag that looks like f(x). Function notation f(x) represents the output value of the function f for the input value x. 

This helps us know exactly what our variables are doing:
*   The variable x in the notation f(x) is the independent variable. This is like the joystick on your controller. You get to decide how to move it.
*   The value f(x) in function notation represents the dependent variable. This is your character's score on the screen. The score depends entirely on how you move the joystick.

When we use math in real life, we change the letters to match the story. In real-world contexts, function notation like D(t) explicitly defines the dependent variable D (like Distance) and the independent variable t (like time). If you see D(5) = 120, you instantly know that at 5 seconds, the runner has gone 120 meters!

### Evaluating Functions
Evaluating a function involves substituting a specific input value or algebraic expression for the independent variable in the function rule. 

Let's evaluate a function together! If our rule is f(x) = 2x + 3, let's find the answer for an input of 4.

Step 1: Write down the original function rule.
f(x) = 2x + 3

Step 2: Swap out the independent variable (x) and substitute it with the number 4.
f(4) = 2(4) + 3

Step 3: Multiply the 2 and the 4 together.
f(4) = 8 + 3

Step 4: Add the 8 and the 3 together.
f(4) = 11

Sometimes, math rules change depending on where you are, like changing levels in a video game! These are called piecewise functions. Evaluating a piecewise function requires determining which domain interval contains the given input value before applying the corresponding mathematical rule. You just have to check which "level" you are in first!

Later, you will use a special formula called a difference quotient. The difference quotient is an algebraic expression evaluated using function notation to represent the average rate of change between two points. It is basically a way to measure how fast your score is going up or down.

## Establishing Boundaries: Finding the Domain

Why do we need to know the domain (the valid inputs)? Because putting the wrong number into a math machine can break the rules of the universe! Finding the domain is like learning where your character is not allowed to walk on the game map.

Here are the strict rules you need to know:
1.  **Polynomials** (normal math formulas without weird fractions or roots): The domain of any polynomial function with real coefficients is the set of all real numbers. You can plug in anything!
2.  **Rational Functions** (fractions with x on the bottom): You can never divide a pizza among zero people! The domain of a rational function excludes any input values that cause the denominator expression to equal zero.
    Let's find the domain of f(x) = 1 / (x - 3).
    Step 1: Look at the denominator (the bottom part of the fraction). Our denominator is x - 3.
    Step 2: Since we can never divide by zero, set the denominator equal to zero to find the danger zone.
    x - 3 = 0
    Step 3: Solve for x by adding 3 to both sides.
    x = 3. 
    Because x = 3 makes the bottom zero, the number 3 is kicked out of our domain!
3.  **Radical Functions with an Even Index** (like a square root): The domain of a radical function with an even index is restricted to input values that make the radicand non-negative. The radicand is the stuff inside the root, and you can't take the square root of a negative number. 
4.  **Radical Functions with an Odd Index** (like a cube root): The domain of a radical function with an odd index includes the set of all real numbers. It works perfectly fine with negatives!
5.  **Logarithmic Functions** (written as "log" or "ln"): The domain of a logarithmic function is restricted to input values that make the argument expression strictly greater than zero.

### Communicating the Domain
Once we know our allowed inputs, we have to write them down using special math grammar. 

Interval notation is one way to do this. Interval notation uses parentheses to indicate that an endpoint is excluded from a domain or range set. (Like saying "everything up to the number 10, but not exactly 10"). On the other hand, interval notation uses square brackets to indicate that an endpoint is included in a domain or range set. (Like saying "everything up to the number 10, and we are including the 10!").

We can also use set-builder notation. Set-builder notation specifies the algebraic properties that elements must satisfy to belong to the domain or range. Think of it like writing a VIP checklist that numbers have to pass to get into a party!

## Defining the Output: Finding the Range

Finding the range (all possible outputs) is like trying to find the lowest valley and the highest mountain in an entire video game world. The range of a function can be determined by analyzing the global minimum and global maximum values on the function's graph. 

Some basic math shapes have strict, built-in limits for their range:
*   **Quadratics:** A parabola is shaped like a giant letter U. The range of a quadratic function f(x) = ax^2 + bx + c is bounded either above or below by the y-coordinate of the parabola's vertex (the absolute bottom or top point of the U).
*   **Absolute Value:** Absolute value bars turn everything inside them positive! So, the range of the basic absolute value function f(x) = |x| includes only non-negative real numbers.
*   **Exponential:** Exponential graphs curve upward super fast. The range of the basic exponential function f(x) = a^x, where a is a positive constant not equal to 1, includes only strictly positive real numbers.

Finally, when we use math in the real world, common sense steps in! Real-world scenarios impose practical domain restrictions based on physical limitations, such as restricting time or distance variables to non-negative values. (You can't play basketball for negative 5 minutes!). In the exact same way, real-world scenarios impose practical range restrictions based on logical limits, such as a maximum physical capacity or prohibiting fractional counts of distinct objects. (If you are counting basketballs, you can't have an output of 3.5 basketballs. It has to be a whole number!)
