Pure anarchy. But a function? A function is a specific type of relation where each input value maps to exactly one output value. It is the sacred rule of reality TV monogamy! If contestant three hands a rose to contestant five, they cannot also hand a rose to contestant nine. One input, exactly one unique output. If an input gets greedy and tries to map to two different outputs, the contract is breached, the determinism is shattered, and the whole mathematical model burns to the ground. 

As the producer of this show, you have to define the cast and the prize pool. The set of all valid input values for a relation is called the domain. These are the people legally allowed on set. The set of all possible output values for a relation is called the range. This is the entire catalog of emotional breakdowns or prizes they can walk away with. When your students are staring at a Cartesian coordinate system, tell them to physically trace the stage: the domain of a function can be found by identifying the horizontal spread of the function's graph, literally scanning left to right to see where the actors are allowed to stand. Meanwhile, the range of a function can be found by identifying the vertical spread of the function's graph, looking up and down to see the absolute highest highs and lowest lows of the season’s drama.

But how do you spot a cheater? How do you know when a relation is trying to masquerade as a function? You have to audit the paperwork. If you are looking at the stage manager's table of values, a table of values represents a function if no single input value is paired with multiple different output values. If you see the number seven listed as an input producing a four, and two rows down that same seven is producing a nine, you flip the table! On the producer's conspiracy string board—a mapping diagram—a relation is a function if exactly one arrow originates from each element in the domain. I don't care if a dozen arrows converge on the same output—that just means everyone has a crush on the same guy—but exactly one arrow must leave each input. 

When you move to the visual edit, you use the security laser grid. The Vertical Line Test states that a graph represents a function if no vertical line intersects the graph more than once. Because a vertical line represents one specific input, if the laser hits the graph twice, you’ve caught one input trying to live a double life. And the ultimate criminal? The vertical line equation $x = c$, where $c$ is a constant, does not represent a function of $x$. It is the ultimate diva, sitting at a single input and demanding an infinite number of outputs. Kick it off the set! 

If you are digging through the raw legal contracts—the algebra—an equation represents $y$ as a function of $x$ if the equation can be solved to yield a unique $y$-value for every valid $x$-value. If you ever see an equation containing an even power of $y$, like $y^2 = x$, sound the alarms! When you take the square root to solve it, you spawn the dreaded "plus or minus" evil twin. You get two outputs for one input, meaning equations containing an even power of $y$ generally do not represent $y$ as a function of $x$.

Once you’ve enforced the rules, you have to write the script. Function notation $f(x)$ represents the output value of the function $f$ for the input value $x$. The variable $x$ in the notation $f(x)$ is the independent variable; it is the wild card contestant we blindly pull from the streets. The value $f(x)$ in function notation represents the dependent variable; it is the resulting drama that depends entirely on who we threw into the room. And evaluating a function? That’s yelling "Action!" Evaluating a function involves substituting a specific input value or algebraic expression for the independent variable in the function rule.

When you take this show into reality, the script changes to match the setting. In real-world contexts, function notation like $D(t)$ explicitly defines the dependent variable $D$ and the independent variable $t$. Distance depending on time! And speaking of the real world, the legal department is going to force you to restrict your inputs and outputs. Real-world scenarios impose practical domain restrictions based on physical limitations, such as restricting time or distance variables to non-negative values because we haven't invented time travel. Likewise, real-world scenarios impose practical range restrictions based on logical limits, such as a maximum physical capacity or prohibiting fractional counts of distinct objects. You cannot output half of a live teenager! 

But the algebra itself has booby traps, and you have to act as the bouncer to the domain. If you're dealing with polynomials, pop the champagne. The domain of any polynomial function with real coefficients is the set of all real numbers. Everyone gets into the club. But the domain of a rational function excludes any input values that cause the denominator expression to equal zero. That is the trap door! If the denominator hits zero, the whole studio implodes. 

What about radicals? The domain of a radical function with an even index is restricted to input values that make the radicand non-negative. That's the VIP room; positive vibes and zeroes only. But the domain of a radical function with an odd index includes the set of all real numbers. It’s the grunge club next door; they don't care if you're negative. Finally, the domain of a logarithmic function is restricted to input values that make the argument expression strictly greater than zero. Not even zero is allowed in there. Strictly positive!

When you publish the guest list, you use standard syntax. Interval notation uses parentheses to indicate that an endpoint is excluded from a domain or range set—giving the number a stiff arm to keep it out. Conversely, interval notation uses square brackets to indicate that an endpoint is included in a domain or range set, wrapping the number in a legally binding bear hug. If you want to describe the guest list by their traits, set-builder notation specifies the algebraic properties that elements must satisfy to belong to the domain or range. 

Finding the range is like predicting the season finale ratings. The range of a function can be determined by analyzing the global minimum and global maximum values on the function's graph. Certain classic mathematical personalities have strict, predictable ranges. The range of a quadratic function $f(x) = ax^2 + bx + c$ is bounded either above or below by the $y$-coordinate of the parabola's vertex—it hits rock bottom or a glass ceiling and bounces back. The range of the basic absolute value function $f(x) = |x|$ includes only non-negative real numbers, stripping away all negativity. And the hype train?
