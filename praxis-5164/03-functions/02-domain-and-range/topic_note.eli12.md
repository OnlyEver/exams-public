Think of a math function like a video game console or a vending machine. You put something in (like coins or button presses), and you get something out (like a snack or your character jumping on screen).

In math, the domain of a relation is the complete set of possible input values. It is the fuel the machine accepts. When we talk about inputs, we are talking about things you get to control. Because of this, the domain of a function corresponds to the values of the independent variable.

On the flip side, the range of a relation is the complete set of possible output values. It is what the machine gives back to you. The range of a function corresponds to the values of the dependent variable, because the output depends totally on what you put in!

But there is one major rule to make this machine work perfectly. In a function, each input value in the domain is mapped to exactly one output value in the range. Imagine if you pressed the "A" button on your controller, and your character randomly jumped, ducked, or threw a fireball. The controller would be broken! One button press (input) must do exactly one thing (output).

## Decoding the Data: Tables, Pairs, and Mappings

Before looking at squiggly lines on a graph, let's look at simple pairs of numbers. You have probably seen sets of coordinate points, like (2, 5) or (3, 8). In these pairs, the first number is the x-value and the second is the y-value.
* The domain of a relation is typically represented by the x-values in a set of ordered pairs.
* The range of a relation is typically represented by the y-values in a set of ordered pairs.

Sometimes, these pairs are organized into a chart or a table. But watch out! If you see the same number twice in the input or output column, you don't write it twice when listing the domain or range. 
* To determine the domain from a table of values, identify the set of all unique input values listed in the table. 
* To determine the range from a table of values, identify the set of all unique output values listed in the table.

We can also draw a picture called a mapping diagram to show this. Think of a mapping diagram like a map showing airplane flights from one city to another. We use arrows to show where the inputs go.
* In a mapping diagram representing a relation, the domain consists of all values from which an arrow originates.
* In a mapping diagram representing a relation, the range consists of all values to which an arrow points.

## Visualizing Boundaries on a Coordinate Graph

When we put lines on a graph, we are looking at the exact space a function takes up.

> **The Shadow Analogy**
> Imagine shining a flashlight from the top and bottom of a graph, casting a shadow down onto the horizontal x-axis line. To determine the domain from a coordinate graph, measure the horizontal spread of the graph along the x-axis. 
> Now, imagine shining a flashlight from the left and right sides of the graph, casting a shadow onto the vertical y-axis line. To determine the range from a coordinate graph, measure the vertical spread of the graph along the y-axis.

Graphs have special symbols at their ends to tell you exactly where they stop:
* A solid dot on a graph endpoint indicates that the corresponding coordinate is included in the domain or range. (It means "stop right exactly on this spot!")
* An open circle on a graph endpoint indicates that the corresponding coordinate is excluded from the domain or range. (It means "get super close to this spot, but don't touch it.")
* An arrowhead on a graph line indicates that the function continues indefinitely in the direction of the arrow. (It goes on forever!)

## The Language of Sets: Notation

When we write down what the boundaries are, we use special math codes. Here are the two main ways to write them.

### Set-Builder Notation
Set-builder notation uses a formal set structure, such as {x | x > 0}, to define a domain or range. You just read that out loud as "All the x numbers, as long as x is bigger than zero." It is just a fancy way to put math rules inside curly brackets.

### Interval Notation
Interval notation is a shortcut that just lists the starting and ending points. 
* Interval notation uses square brackets to indicate that an endpoint is included in the domain or range. (Like [2, 5], which means 2 and 5 are included. This matches the solid dot!).
* Interval notation uses parentheses to indicate that an endpoint is excluded from the domain or range. (Like (2, 5), meaning we get close but don't include them. This matches the open circle!).
* Because you can never actually "reach" infinity, in interval notation, the infinity symbol and negative infinity symbol are always enclosed by a parenthesis. 

## The Algebraic Blueprint: Function Families

In math, equations come in different "families." Each family behaves a bit differently.

### The Unrestricted Functions
Some functions will accept any number you want to give them!
* **Linear:** A straight, diagonal line goes on forever. The domain of a linear function without specific constraints is the set of all real numbers. Also, the range of a non-horizontal linear function without specific constraints is the set of all real numbers.
* **Polynomial:** This is a big family of smooth, curving lines. Just like straight lines, the domain of a polynomial function without specific constraints is the set of all real numbers.

### The Stubborn Flatliners
Imagine a straight line that is perfectly flat, like y = 4. It never goes up or down. 
* A constant function of the form y = c has a domain of all real numbers. You can plug in any x value you want.
* But because it never goes up or down, a constant function of the form y = c has a range consisting solely of the single value c. (The output is always just 4!)

### Functions with Floors and Ceilings
Some graphs bounce or change direction, which means they have a lowest point (a floor) or a highest point (a ceiling).
* **Absolute Value:** This function turns everything positive because it measures distance (and distance is never negative!). So, the range of an absolute value function of the form y = |x| is the set of all non-negative real numbers.
* **Quadratics:** These graphs are shaped like a U or an upside-down U, called a parabola. The top or bottom tip of this curve is called the vertex. The range of a quadratic function is restricted by the y-coordinate of the parabola's vertex. 
  * For a quadratic function whose parabola opens upwards, the range is all real numbers greater than or equal to the minimum y-value. 
  * For a quadratic function whose parabola opens downwards, the range is all real numbers less than or equal to the maximum y-value. 

### Functions with Fragile Domains
Some equations have hidden traps. If you put the wrong number in, the math breaks! We have to find these traps to figure out the domain.
* **Rational Functions (Fractions):** You can never divide by zero. It breaks the rules of math. Because of this, the denominator of a fraction in an algebraic function cannot be equal to zero. To find the domain of a rational function, exclude any input values that cause the denominator to equal zero.
  * *Example: Find the domain of y = 10 / (x - 2)*
  * **Step 1:** Look at the bottom part of the fraction (the denominator). It is x - 2.
  * **Step 2:** Set the denominator equal to zero to find the trap: x - 2 = 0.
  * **Step 3:** Add 2 to both sides to solve for x. This gives us x = 2.
  * **Step 4:** State the domain. The domain is all real numbers *except* 2.
* **Square Root Functions:** You cannot take the square root of a negative number. The expression inside a square root must be greater than or equal to zero for the function to output a real number. To find the domain of a square root function, set the radicand greater than or equal to zero and solve for the input variable. (The "radicand" is just a fancy word for the math inside the square root symbol).
  * *Example: Find the domain of y = square root of (x + 5)*
  * **Step 1:** Look at the expression inside the square root. It is x + 5.
  * **Step 2:** Set that expression greater than or equal to zero: x + 5 >= 0.
  * **Step 3:** Subtract 5 from both sides to solve for x. This gives us x >= -5.
  * **Step 4:** State the domain. The domain is all real numbers greater than or equal to -5.

### Algebraic Range Finding
Finding the domain by looking at the x part of the equation is usually easy. Finding the range requires a cool trick. To determine the range of a function algebraically, solve the equation for the independent variable and find the domain of the resulting expression. This means you just shuffle the equation around to get x by itself, and then look for traps again!
  * *Example: Find the range of y = 8 / x*
  * **Step 1:** We start with y = 8 / x. We need to get x by itself.
  * **Step 2:** Multiply both sides by x to move it out of the fraction. This gives us: x * y = 8.
  * **Step 3:** Divide both sides by y to get x completely alone. This gives us: x = 8 / y.
  * **Step 4:** Look for traps in the new equation! We have y on the bottom of a fraction. The denominator cannot be zero.
  * **Step 5:** State the range. The range is all real numbers *except* 0.

## The Physical World: Practical Domain and Range

Math isn't just about floating numbers; it's about the real world! When we use math to measure real things like time, money, or sports, we have to use common sense.

### Continuous vs. Discrete Variables
First, we need to know if our numbers can be chopped up into pieces.
* A continuous domain represents an unbroken interval of real numbers. This means you can keep zooming in and finding smaller and smaller numbers, like decimals and fractions. A function modeling continuous quantities like time or distance typically has a continuous domain. If a soccer game takes 2 hours, you also experienced 1.5 hours, and 1.53 hours!
* A discrete domain consists of distinct, separate input values rather than an unbroken interval. You cannot chop these things in half. A function modeling the count of indivisible physical objects must have a discrete domain consisting of non-negative integers. For example, if you are ordering pizzas with your \$15 allowance, you can buy 2 pizzas or 3 pizzas, but you cannot order 2.15 pizzas!

### Logic and Context
Finally, we use our brains to set real-world boundaries. 
* The practical domain is the set of input values that logically make sense in the real-world context of a problem. 
* The practical range is the set of output values that result from evaluating the practical domain in a real-world scenario.

Let's look at how the real world changes our math rules:
1. **Time:** A regular line on a graph can go into negative numbers forever. But if your x-axis represents a stopwatch ticking during a basketball game, you can't have negative time. A real-world context involving time elapsed since an event cannot have negative input values in the practical domain. The stopwatch starts at zero!
2. **Measurements:** If your y-axis represents the height of a bouncing tennis ball, the math equation might plunge into negative numbers. But the ball hits the floor; it doesn't fall through the center of the earth! A real-world context measuring physical dimensions cannot include negative values in the practical range. The floor is zero!
